# MindMap Directed Connector Implementation Plan

> Implements the approved design in
> `docs/superpowers/specs/2026-07-16-mindmap-directed-connectors-design.md`.
> Work from `decoded/mindmap.html`, then rebuild `tools/mindmap.html`.

## 1. Persist arrow mode compatibly

1. Give every new connector a normalized `data-arrow` value of `none`.
2. Include the value in connector snapshots for history, autosave, ZIP/JSON,
   and standalone HTML.
3. Default missing or unrecognized imported values to `none` when loading.
4. Ensure connectors generated by split/branch operations begin plain.

## 2. Render edge-to-edge connector geometry

1. Add a bounded helper that turns two Node centre points and their rectangles
   into visible start/end intersections.
2. Use it from `updateLine` for horizontal, vertical, diagonal, and
   near-coincident Nodes without invalid SVG values.
3. Use the resulting visible midpoint for the existing branch `+` and all
   branch/split junction placement.

## 3. Add arrowhead control and SVG markers

1. Add reusable SVG marker definitions after SVG reset/reload.
2. Add a hover-only arrow-mode button adjacent to each connector midpoint
   control; retain the current `+` behavior unchanged.
3. Cycle `none → forward → reverse → both → none` and save one history state
   per click.
4. Apply start/end markers based on `data-arrow`, with icons/tooltips that
   communicate each current state.
5. Remove both controls whenever a connector is deleted or invalidated.

## 4. Integrate and verify

1. Confirm theme, PNG, HTML, ZIP, legacy JSON, backup, Undo/Redo, node move,
   and zoom-to-fit preserve normal rendering.
2. Run JavaScript syntax checks on source and generated page, `node
   build-tools.js`, and `git diff --check`.
3. Ask the user to refresh the local MindMap and test geometry, four arrow
   modes, branching, persistence, and exports.
